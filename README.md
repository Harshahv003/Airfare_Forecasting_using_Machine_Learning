## 📌 Project Overview

**FlightFare AI** is a production-ready airfare price prediction system built with machine learning. It predicts Indian domestic flight ticket prices based on airline, route, travel date, departure time, and number of stops.

The system was trained on **10,682 real flight records** and achieves an **R² score of 0.84** using a Gradient Boosting model — meaning it explains 84% of the variance in airfare pricing.

This project was built for:
- **Final Year Academic Submission** (ML + Flask full-stack)
- **Freelance Portfolio** (client-level code quality and UI)

## ⚙️ Installation & Setup

### Prerequisites
- Python 3.11+
- pip

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/airfare-prediction.git
cd airfare-prediction
```

### 2. Create virtual environment

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Add the dataset

Place `Data_Train.xlsx` in the `dataset/` folder.

### 5. Train the model

```bash
python model/train_model.py
```

This will:
- Clean and engineer features
- Train Linear Regression, Random Forest, and Gradient Boosting
- Run hyperparameter tuning
- Save the best model to `model/flight_price_model.pkl`
- Generate 8 EDA + evaluation plots in `static/plots/`

### 6. Run the app

```bash
python app.py
```

Open: **http://127.0.0.1:5000**

## 🔌 API Reference

### `POST /predict`

Predict the price of a flight.

**Request body (JSON):**
```json
{
  "airline":          "IndiGo",
  "source":           "Delhi",
  "destination":      "Cochin",
  "journey_date":     "2024-05-15",
  "dep_time":         "06:00",
  "arrival_time":     "10:30",
  "stops":            1,
  "duration_minutes": 270
}
```

**Response:**
```json
{
  "success":    true,
  "price":      8750.50,
  "price_low":  7875.45,
  "price_high": 9625.55,
  "error":      null,
  "meta": {
    "name":  "Gradient Boosting (XGBoost-style)",
    "r2":    0.8396,
    "mae":   1212.83,
    "rmse":  1859.56
  }
}
```

---

### `GET /api/meta`

Returns model info and all supported input values.

```json
{
  "model":        { "name": "...", "r2": 0.84, "mae": 1212, "rmse": 1859 },
  "airlines":     ["Air Asia", "Air India", ...],
  "sources":      ["Banglore", "Chennai", ...],
  "destinations": ["Banglore", "Cochin", ...],
  "stops":        [0, 1, 2, 3, 4]
}
```

---

### `GET /api/trends?type=monthly`

Returns price trend data for charting.

| Param | Values | Description |
|---|---|---|
| `type` | `monthly`, `airline`, `route` | Trend grouping |
| `source` | City name | Optional filter |
| `destination` | City name | Optional filter |

---

### `GET /health`

Health check for deployment monitors.

```json
{ "status": "ok", "model_loaded": true }
```

## 🧠 ML Pipeline Details

### Dataset
- **10,682** flight records
- **11 features** → engineered to **13 model features**
- Date range: March–June 2019 (Indian domestic flights)

### Feature Engineering

| Raw Feature | Engineered Feature |
|---|---|
| `Date_of_Journey` | `Journey_Day`, `Journey_Month` |
| `Dep_Time` | `Dep_Hour`, `Dep_Minute`, `TimeSlot_enc` |
| `Arrival_Time` | `Arrival_Hour`, `Arrival_Minute` |
| `Duration` | `Duration_Minutes` |
| `Total_Stops` | `Stops` (integer 0–4) |
| `Route` | `Route_Hops` |
| `Airline` | `Airline_enc` (LabelEncoder) |
| `Source` | `Source_enc` |
| `Destination` | `Destination_enc` |

### Model Comparison

| Model | R² | MAE | RMSE |
|---|---|---|---|
| Linear Regression (baseline) | 0.4381 | ₹2,448 | ₹3,481 |
| Random Forest (default) | 0.7963 | ₹1,186 | ₹2,096 |
| Gradient Boosting | **0.8396** | **₹1,213** | **₹1,860** |
| Random Forest (tuned) | 0.8217 | ₹1,219 | ₹1,960 |

### Key Findings from EDA
- **Stops** have the strongest correlation with price (r=0.60)
- **Duration** is second strongest (r=0.51)
- **Jet Airways Business** is 3× more expensive than economy average
- **Non-stop flights** average ₹5,025 vs ₹17,686 for 4-stop flights
- Prices peak in **May–June** (summer travel season)

## 🚀 Deployment Guide

### Deploy to Render (Free tier)

1. Push your code to GitHub
2. Go to [render.com](https://render.com) → New Web Service
3. Connect your GitHub repo
4. Set:
   - **Build Command:** `pip install -r requirements.txt && python model/train_model.py`
   - **Start Command:** `gunicorn app:app --bind 0.0.0.0:$PORT --workers 2`
5. Add dataset file via Render's file upload or use a pre-trained model

### Deploy to Railway

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login and deploy
railway login
railway init
railway up
```
